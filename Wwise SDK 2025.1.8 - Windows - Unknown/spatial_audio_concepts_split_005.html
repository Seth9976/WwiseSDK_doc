Combining Acoustic Methodologies

When building your projects, it is important to understand the differences between the two sound propagation models:

  • The Wwise Acoustics system simulates sound propagation automatically based on geometry, which you pass to it with the AK::SpatialAudio::SetGeometry API. It models direct and diffuse sound in parallel (it uses Rooms and Portals to model diffuse sound), and all propagation paths can be affected by physical phenomena. Wwise Acoustics drives Diffraction and Transmission Loss, which Wwise maps to properties (Volume, LPF, HPF, and DSF) through curves.
  • The Wwise sound engine also models sound propagation. It does not use a simulation, but instead relies on obstruction and occlusion values you provide through APIs such as AK::SoundEngine::SetObjectObstructionAndOcclusion. Wwise maps obstruction and occlusion values to properties (Volume, LPF, HPF, and DSF) through curves.

In short, Wwise Acoustics uses a simulation to automatically derive Diffraction and Transmission loss values from geometry, while the sound engine processes explicit obstruction and occlusion values set by API. In both cases, Wwise maps the values to properties.

Each methodology has its uses, and you can use both approaches in the same project. However, if you do so, you must not use them on the same local segment of a propagation path. For example:

  • An actor might obstruct a path segment, with an obstruction value set through AK::SoundEngine::SetObjectObstructionAndOcclusion. The next segment of the path might then diffract at a Portal edge, at which point Wwise Acoustics manages the sound propagation.
  • Wwise Acoustics computes Portal diffraction, but you can use AK::SpatialAudio::SetPortalObstructionAndOcclusion to set obstruction or occlusion values on an additional object placed in or in front of the Portal but not included in the geometry.
