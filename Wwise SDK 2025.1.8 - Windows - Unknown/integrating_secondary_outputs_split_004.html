Examples

All code examples below assume the following declarations:

// The examples below assume these game object IDs are already registered.
const AkGameObjectID POW_GAME_OBJECT = 100; // Emitter game object for the "Pow!" sound
const AkGameObjectID PIF_GAME_OBJECT = 150; // Emitter game object for the "Pif!" sound
const AkGameObjectID MY_LISTENER1 = 200; // Listener game object for the first player's controller speaker
const AkGameObjectID MY_LISTENER2 = 300; // Listener game object for the second player's controller speaker
const AkGameObjectID MY_MAIN_LISTENER = 400; // MY_MAIN_LISTENER is the default listener you normally use, defined by AK::SoundEngine::SetDefaultListeners.
// Platform-specific IDs for the controller speaker devices.
// To resolve these, on Windows you can use AK::GetDeviceIDFromName.
AkUniqueID firstPlayerDeviceID;
AkUniqueID secondPlayerDeviceID;

The code samples below apply to all platforms. The only platform-specific bit concerns how firstPlayerDeviceID and secondPlayerDeviceID are obtained. Refer to Targeting Specific System Devices to learn how to resolve device IDs for your target platform.

Sound on one game controller output:

// Add a secondary output connected to the speaker endpoint for player 0.
AkOutputSettings outputSettings("Game Controller Speaker", firstPlayerDeviceID);
AK::SoundEngine::AddOutput(outputSettings);
// Play an Event. In the Wwise project, this sound must be routed to a master bus that is set to output in "Game Controller Speaker" (or any sub-bus).
AK::SoundEngine::PostEvent("Play_Pow", POW_GAME_OBJECT);

Same sound on two different controllers:

AkGameObjectID listenerArray1[] = { MY_LISTENER1 };
AkGameObjectID listenerArray2[] = { MY_LISTENER2 };
AkGameObjectID listenerArrayBoth[] = { MY_LISTENER1, MY_LISTENER2 };
// Add a secondary output connected to the speaker output for player 0, associated with listener #1.
AkOutputSettings outputSettings1("Game Controller Speaker", firstPlayerDeviceID);
AK::SoundEngine::AddOutput(outputSettings1, NULL, listenerArray1, 1);
// Add a secondary output connected to the speaker output for player 1, associated with listener #2.
AkOutputSettings outputSettings2("Game Controller Speaker", secondPlayerDeviceID);
AK::SoundEngine::AddOutput(outputSettings2, NULL, listenerArray2, 1);
// Set a game object to emit sound to the listener 1(player 0) and 2 (player 1).
AK::SoundEngine::SetListeners(POW_GAME_OBJECT, listenerArrayBoth, 2);
// Play an Event. In the Wwise project, this sound must be routed to a master bus that is set to output in "Game Controller Speaker" (or any sub-bus).
AK::SoundEngine::PostEvent("Play_Pow", POW_GAME_OBJECT);

Different sounds on two different controllers:

AkGameObjectID listenerArray1[] = { MY_LISTENER1 };
AkGameObjectID listenerArray2[] = { MY_LISTENER2 };
// Add a secondary output connected to the speaker output for player 0, associated with listener #1.
AkOutputSettings outputSettings1("Game Controller Speaker", firstPlayerDeviceID);
AK::SoundEngine::AddOutput(outputSettings1, NULL, listenerArray1, 1);
AkOutputSettings outputSettings2("Game Controller Speaker", secondPlayerDeviceID);
AK::SoundEngine::AddOutput(outputSettings2, NULL, listenerArray2, 1);
// Set a game object to emit sound to the proper listeners.
AK::SoundEngine::SetListeners(POW_GAME_OBJECT, listenerArray1, 1);
AK::SoundEngine::SetListeners(PIF_GAME_OBJECT, listenerArray2, 1);
// Play an Event. In the Wwise project, this sound must be routed to a master bus which is set to output in "Game Controller Speaker" (or any sub-bus).
AK::SoundEngine::PostEvent("Play_Pow", POW_GAME_OBJECT);
AK::SoundEngine::PostEvent("Play_Pif", PIF_GAME_OBJECT);

Same sound on one game controller output and on the TV:

// Add a secondary output connected to the speaker output for player 0, associated with listener #1.
AkOutputSettings outputSettings1("Game Controller Speaker", firstPlayerDeviceID);
AkGameObjectID listenerArray1[] = { MY_LISTENER1 };
AK::SoundEngine::AddOutput(outputSettings1, NULL, listenerArray1, 1);
// Set a game object to emit sound to the listener 0 (TV) and 1 (controller for first player).
AkGameObjectID listenerArrayWithDefault[] = { MY_MAIN_LISTENER, MY_LISTENER1 };
AK::SoundEngine::SetListeners(POW_GAME_OBJECT, listenerArrayWithDefault, 2);
// Play an Event.
// In the Wwise project, this sound must be routed to a master bus that is set to output in "Game Controller Speaker" (or any sub-bus).
// Also, this sound must have an Auxiliary send routed to the Main Audio Bus.
AK::SoundEngine::PostEvent("Play_Pow", POW_GAME_OBJECT);

You can find a working example of multiple output management in the Integration Demo Sample, in the DemoMotion page (multi-player), and in the BGMDemo page (DVR/BGM management).
