Game Setup

On the game side, link with the separate library called AkMotionSink. This library provides support for the standard controllers of supported platforms. You also need to include the AkMotionSinkFactory.h file, located under SDK\include\AK\plugin. It is important to include this file because it automatically registers the plug-in.

Note: In Unity and Unreal, plug-in libraries are managed automatically. You don't have to add AkMotionSink manually.

Here are the supported controllers and additional requirements by platform:

  • Windows
    • Device: XInput-compatible controllers
    • Device channel config and layout: Anonymous 2-channel (left motor, right motor)
    • Additional requirements:
      • Xinput.lib
  • Windows
    • Device: GameInput-compatible controllers
    • Device channel config and layout: Anonymous 4-channel and Stereo 2-channel
    • Additional requirements: See Support for GameInput on Windows for more information.
  • Windows
    • Device: DualShock 4 and DualSense Controllers
    • Device channel config and layout: Anonymous 4-channel and Stereo 2-channel
    • Additional requirements: See "Support for DualShock 4 and DualSense on Windows using libScePad" on this page for more information (if viewing this documentation on the Audiokinetic website, you must be logged in and have a Playstation 5 license).
  • Windows (OpenXR – PC VR with Meta Horizon Link)
    • Device: Meta Quest Touch controllers connected through Meta Horizon Link
    • Device channel config and layout: Stereo 2-channel (left controller, right controller)
    • Additional requirements: See Support for Meta Quest through OpenXR for more information.

If your application uses motion with one or more devices, you must add a dedicated output for each device. For example, a split-screen game with four players connected needs four different outputs for the controllers to receive haptic feedback. To add an output device, use the Wwise API's AK::SoundEngine::AddOutput function and specify the ShareSet name (as defined in your Wwise project) in the AkOutputSettings parameter. Additionally, because multiple devices can be connected, you need to provide a device ID. The following list contains more information about device IDs.

  • Windows
    • Device: XInput-compatible controllers
    • Information: Use a player index between 0 to 3.
  • Windows
    • Device: GameInput-compatible controllers
    • Information: Retrieve the GameInputDeviceInfo for a gamepad, and call AK::GetAppLocalDeviceIdHash with the address of the GameInputDeviceInfo::deviceId as the parameter.
      The Motion output is only initialized for devices with either a non-zero value for GameInputDeviceInfo::supportedRumbleMotors or a non-zero value for GameInputHapticInfo::locationCount.
      Devices that only use force feedback, such as some racing wheels and flight sticks, are not supported.
  • Windows (OpenXR – PC VR with Meta Horizon Link)
    • Device: Meta Quest Touch controllers connected through Meta Horizon Link
    • Information: Use 0. After OpenXR functions are initialized through Support for Meta Quest through OpenXR, a device ID of 0 routes to the OpenXR haptic sink instead of the default XInput device.
Note: On all platforms except Windows, a device ID of 0 targets the first available device that supports motion. On Windows, a device ID of 0 is initialized as an XInput device.

Remember that game controllers can be disconnected either physically or due to communication problems. Disconnection does not have any adverse effects on the sound engine other than needlessly using resources. If you think a device has been disconnected for a long time, call AK::SoundEngine::RemoveOutput and provide the AkOutputDeviceID returned by the corresponding AddOutput() function call.
