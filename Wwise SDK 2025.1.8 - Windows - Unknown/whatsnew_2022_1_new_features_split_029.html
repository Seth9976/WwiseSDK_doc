Improvements to Custom Scheduling of Audio Rendering Tasks

Multithreaded execution of the audio rendering pipeline has been rewritten and optimized to achieve greater runtime performance. This was most notably achieved by moving to an internal job-based scheduler with modeling of dependencies, instead of a fork-and-join method of parallel execution. Key highlights are:

  • Execution of busses in the Voices Graph has greatly improved concurrency and throughput by allowing for independent branches of the Voice Graph to be processed concurrently, instead of processing only one level of the voice graph at a time.
  • Voice processing now has improved load balancing by default, without requiring the game engine to handle such distribution of work.
  • Integration with a game engine job manager is easier to support and maintain. In most circumstances, the game engine should also be able to schedule fewer jobs than before, reducing the overhead of the integration.
  • New APIs have been added to allow for detailed profiling of Wwise CPU usage in external tools.

This means that Wwise uses CPU resources more effectively to deliver a richer audio experience, while also providing better coordination of work and scheduling with the game engine.
