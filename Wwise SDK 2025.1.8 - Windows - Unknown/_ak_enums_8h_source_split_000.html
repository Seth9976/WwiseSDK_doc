359 /// The AkDynamicSequenceType is specified when creating a new dynamic sequence.
360 ///
361 /// In sample accurate mode, when a dynamic sequence item finishes playing and there is another item
362 /// pending in its playlist, the next sound will be stitched to the end of the ending sound. In this
363 /// mode, if there are one or more pending items in the playlist while the dynamic sequence is playing,
364 /// or if something is added to the playlist during the playback, the dynamic sequence
365 /// can remove the next item to be played from the playlist and prepare it for sample accurate playback before
366 /// the first sound is finished playing. This mechanism helps keep sounds sample accurate, but then
367 /// you might not be able to remove that next item from the playlist if required.
368 ///
369 /// If your game requires the capability of removing the next to be played item from the
370 /// playlist at any time, then you should use the DynamicSequenceType_NormalTransition option instead.
371 /// In this mode, you cannot ensure sample accuracy between sounds.
372 ///
373 /// Note that a Stop or a Break will always prevent the next to be played sound from actually being played.
374 ///
375 /// \sa
376 /// - AkCmd_DynamicSequence_Open
377 /// - AK::SoundEngine::DynamicSequence::Open
378 enum AkDynamicSequenceType

380  AkDynamicSequenceType_SampleAccurate, ///< Sample accurate mode
