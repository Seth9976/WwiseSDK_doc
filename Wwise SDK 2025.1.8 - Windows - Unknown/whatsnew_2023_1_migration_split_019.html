AK::SoundEngine::Suspend No Longer Influences the State of AK::Comm

On some platforms, the AK::SoundEngine::Suspend() and AK::SoundEngine::WakeupFromSuspend() calls used to suspend and wake up the AK::Comm module automatically, and disconnected any connected Wwise Profilers. Now, clients must manage AK::Comm separately. Refer to Handling System-Specific Events to learn how to manage the AK::Comm module in the context of application lifecycle events for your supported platforms.
