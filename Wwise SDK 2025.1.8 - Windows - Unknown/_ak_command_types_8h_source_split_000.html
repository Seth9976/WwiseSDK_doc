138 /// Posts an event. When this command is executed, the actions referenced in the event will be executed.
139 /// The user is responsible for providing a new unique playingID value for every instance of this command.
140 /// Re-using the same playing ID will cause actions in this event to be associated with an existing playback.
141 ///
142 /// If \c numExternalSources is set to a value higher than 0, then the client is expected to call AK_CommandBuffer_AddExternalSources after the command:
143 ///
144 /// auto cmd = (AkCmd_PostEvent*)AK_CommandBuffer_Add(buffer, AkCommand_PostEvent);
145 /// // Fill out the command...
146 /// cmd->numExternalSources = myExternalSourcesArray.size();
147 /// AK_CommandBuffer_AddExternalSources(buffer, myExternalSourcesArray.size(), myExternalSourcesArray.data());
148 ///
149 /// This command can fail for the following reasons:
150 /// - AK_InvalidParameter: \c eventID is not valid or \c numExternalSources is higher than 0 and not enough external sources were added after the command.
151 /// - AK_InsufficientMemory: Failed to allocate memory necessary to begin processing the command
152 /// - AK_ResourceInUse: \c playingID was already used in a previous post event command. Use \c AK_SoundEngine_GeneratePlayingID() to ensure that a unique ID is generated for each post event command.
153 /// - AK_IDNotFound: Event ID not found.
154 /// - AK_PartialSuccess: When connected to the Wwise Profiler, command has been delayed to a later frame until Wwise synchronizes the event.
155 /// - AK_MaxReached: Event Cooldown parameters for eventID did not allow the event to play.
156 ///
157 /// \sa
158 /// - AkCommand_PostEvent
159 /// - AK_CommandBuffer_AddExternalSources
160 struct AkCmd_PostEvent
161 {
162  AkPlayingID playingID; ///<Unique ID that will be associated with this playback. Use \ref AK_SoundEngine_GeneratePlayingID() to generate a new unique playing ID.
163  AkUniqueID eventID; ///<Unique ID of the event
164  AkGameObjectID gameObjectID; ///<Associated game object ID
165  AkPlayingID actionTargetPlayingID; ///<Filters the "Target" of actions in the event by a playing ID. Set to AK_INVALID_PLAYING_ID to disable target filtering. Not all actions are affected by this; see AkActionOnEventType for the list of action types affected by this option.
166  AkUInt32 flags; ///<(optional) Bitmask: see \ref AkCallbackType
167  AK_WPTR(AkEventCallbackFunc, callback); ///<(optional) Callback function
168  AK_WPTR(void*, callbackCookie); ///<(optional) Callback cookie
169  AkUInt32 numExternalSources; ///<(optional) Number of elements in \c externalSources array
170 };

216 /// Sets the value of a real-time parameter control.
217 /// This command may set a game parameter value with global scope, game object scope, or playing ID scope.
218 /// Playing ID scope supersedes both game object scope and global scope, and game object scope supersedes global scope.
219 /// (Once a value is set for a specific playing ID, it will not be affected by changes at the game object scope or global
220 /// scope. Similarly, a value set at the game object scope will not be affected by a global scope change.) Game parameter
221 /// values set with a global scope are applied to all game objects that not yet registered, or already registered but not
222 /// overridden with a value with game object or playing id scope. To set a game parameter value with global scope,
223 /// \c gameObjectID must be set to \c AK_INVALID_GAME_OBJECT \a and \c playingID must be set to \c AK_INVALID_PLAYING_ID.
224 /// To set a game parameter value with object scope, \c gameObjectID must be set to a valid game object ID \a and
225 /// \c playingID must be set to \c AK_INVALID_PLAYING_ID. Finally, to set a game parameter value with playing ID scope,
226 /// both \c gameObjectID \a and \c playingID must be provided.
227 /// With this command, you may also change the value of a game parameter over time. To do so, specify a non-zero
228 /// value for \c transitionTime. At each audio frame, the game parameter value will be updated internally
229 /// according to the interpolation curve.
230 ///
231 /// This command can fail for the following reasons:
232 /// - AK_InvalidParameter: \c gameObjectID specified and is outside the valid range or \c fadeCurve is not a valid AkCurveInterpolation value.
233 /// - AK_IDNotFound: \c gameObjectID is not a registered game object ID.
234 ///
235 /// \sa AkCommand_SetRTPC
236 struct AkCmd_SetRTPC
237 {
238  AkRtpcID rtpcID; ///<Game parameter ID
239  AkRtpcValue rtpcValue; ///<Value to set
240  AkGameObjectID gameObjectID; ///<(optional) Game Object ID; specify AK_INVALID_GAME_OBJECT for global scope or if using playingID
241  AkPlayingID playingID; ///<(optional) Playing ID; specify AK_INVALID_PLAYING for global scope or if using gameObjectID
242  AkTimeMs transitionTime; ///<(optional) Duration during which the game parameter is interpolated towards in_value, 0 for instant change
243  AkUInt16 fadeCurve; ///<(optional) Curve type to be used for the game parameter interpolation, see \ref AkCurveInterpolation
244  AkUInt16 bypassInternalValueInterpolation; ///<Not 0 if you want to bypass the internal "slew rate" or "over time filtering" specified by the sound designer. This is meant to be used when, for example, loading a level and you don't want the values to interpolate.
245 };

387 /// Sets a listener's spatialization parameters. This lets you define listener-specific
388 /// volume offsets for each audio channel.
389 /// If \c isSpatialized is false, the volumes array is used for this listener (3D positions
390 /// have no effect on the speaker distribution). Otherwise, the volumes array is added to the speaker
391 /// distribution computed for this listener.
392 /// \remarks
393 /// - If a sound is mixed into a bus that has a different speaker configuration than \c channelConfig,
394 /// standard up/downmix rules apply.
395 /// - Sounds with 3D Spatialization set to None will not be affected by these parameters.
396 ///
397 /// When \c channelConfig is set to a valid channel config, the Sound Engine expects an array of \c AkReal32 objects after the command.

403 /// AK_CommandBuffer_AddArray(buffer, sizeof(AkReal32), cmd->channelConfig.uNumChannels, volumes);
404 ///

406 ///
407 /// This command can fail for the following reasons:
408 /// - AK_InvalidParameter: \c listenerID is outside the valid range, \c channelConfig is invalid, incomplete volumes array after the command

410 /// - AK_IDNotFound: \c gameObjectID is not a registered game object ID.
411 ///
412 /// \sa
