Fixes for Community-Reported Bugs

  • WG-54889 Fixed: WAAPI does not properly support WebSocket fragments (RPC calls with large payloads fail in Python).
  • WG-55303 Fixed: Poor performance in Listener-Relative Routing on busses with height channels.
  • WG-55786 Fixed: Crash when using AK::SoundEngine::RegisterBusMeteringCallback.
  • WG-55808 Fixed: Crash when changing the Game Parameter that controls a property on a Modulator LFO while the sound is playing.
  • WG-56351 Fixed: Crash in CAkRTPCMgr::AkRTPCSubscription::PushUpdate when Blend Containers are driven by Modulators with properties controlled by RTPCs.
  • WG-56429 Fixed: (Spatial Audio) The transmission loss value for the listener's room is ignored on the direct transmission path for room tones and reverb if there are one or more rooms between the listener and the source.
  • WG-56562 Fixed: WAAPI calls are processed at a very slow rate during the SoundBank pre-generation and post-generation steps.
  • WG-56645 Fixed: XMA crash on GDK when recovering from an earlier decode error.
  • WG-56664 Fixed: Assert and crash when parameters are driven by MIDI Velocity.
  • WG-56680 Fixed: Performance degradation when many Switch Containers are loaded and when the values of RTPC-driven Switches change.
  • WG-56717 Fixed: Possible erroneous assert in function CAkParameterTarget::SetRootNode.
  • WG-56794 Fixed: Crash when entering a portal.
  • WG-56865 Fixed: Crash when profiling an Audio Device Effect such as the Mastering Suite.
  • WG-57117 Fixed: (Spatial Audio) When a Portal connects two rooms, Game Objects are sometimes reported as being in the wrong room.
  • WG-57204 Fixed: Crash when using XMA and unloading banks through memory pointers at the same time, instead of using the bank ID or file name.
