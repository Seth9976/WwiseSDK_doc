Using IDs or Strings (Unicode or ANSI)

The SDK provides two ways of accessing items inside a bank: using strings or using IDs. There are a few things that you should consider before deciding which option to use in your game, including:

  • The importance of having reader-friendly code
  • The use of strings in other areas of your development environment
  • CPU usage
  • Memory usage

Using Strings

  • Makes code more readable, which makes it suitable for development.
  • Appropriate for development environments that normally use strings.
  • A small amount of CPU is required to convert strings to the IDs used by sound engine. However, most games will not notice the difference in CPU usage unless thousands of stings are converted per game frame.
  • Strings are not stored in the sound engine, so if the strings are already used in the game engine, then no additional memory will be used. If, however, the game doesn't use strings, then some memory will be used on the game side to store the strings before they are converted.

Using IDs

  • Code is less readable, which makes it more difficult to troubleshoot problems during development.
  • Doesn't require any additional CPU usage because the sound engine uses IDs internally.
  • Doesn't require any additional memory.

In most cases, you can get away with using strings; but, if your game is very squeezed in terms of memory and CPU, you might want to consider using IDs.
