WAAPI

The following features were added to WAAPI:

  • Added to ak.wwise.core.object.set :
    • Batch operations on Wwise objects.
    • Ability to set properties and references.
    • Ability to create new objects as children or inside a list (for example, RTPCs).
    • RTPCs can be now added, modified, and removed.
    • Custom and ShareSets Modulators can now be created.
    • Custom and ShareSets Effects can now be created.
    • Source plug-ins can now be created.
  • Added ak.soundengine.getState and ak.soundengine.getSwitch to retrieve the current Switch and current State from the sound engine.
  • Added ak.wwise.core.soundbank.convertExternalSources to support external source conversion.
  • Added ak.wwise.core.soundbank.processDefinitionFiles to support SoundBank definition files.
  • Added topics ak.wwise.core.profiler.gameObjectRegistered and ak.wwise.core.profiler.gameObjectUnregistered to be notified when a game object is registered or unregistered.
  • Added ak.wwise.core.profiler.getGameObjects to obtain the list of game objects registered at a specific moment in a profiler capture.
  • The Mute and Solo commands now accept true/false values being passed, and automatically toggle when no state is specified.
  • Added topic ak.wwise.core.profiler.captureLog.itemAdded to be notified of new Capture Log Events published (also added in 2021.1.9+).
  • Added ak.wwise.core.object.diff and ak.wwise.core.object.pasteProperties to support the Paste Properties features (refer to Paste Properties).
  • Added ak.wwise.core.getProjectInfo to retrieve information about the currently opened project.
  • Added ak.wwise.core.undo.undo to undo the last operation.
  • Added ak.wwise.core.sound.setActiveSource to change the active Source of a Sound object.
  • Added ak.wwise.core.transport.prepare to prepare the object and its dependencies for playback.
