API Changes

  • WG-21648 Added AK::SoundEngine::GetSourcePlayPositions() function to obtain position of all sounds triggered by an event.
  • WG-27609 Reviewed the plug-in registration model and SDK to allow dynamic linking of plug-ins.
  • WG-28215 Made GetDownstreamGain() plug-in service available to all types of plug-ins.
  • WG-28254 Added query function AK::SoundEngine::Query::GetRTPCValueByPlayingID().
  • WG-28259 Added AK::Wwise::IAudioPlugin::GetSourceDuration() method.
  • WG-28434 Updated the migration notes.
  • WG-28482 Removed the MP3 source plug-in sample.
  • WG-28507 Moved music grid timings into AkSegmentInfo, and made the latter available from music sync callbacks as well as from AK::MusicEngine::GetPlayingSegmentInfo().
  • WG-28509 Removed the uMaxNumTransitions parameter from the AkInitSettings structure. The maximum number of transitions can now freely grow as needed. There is no more transition stealing system and all transitions will now be honored unless the system runs out of memory. It is the game's responsibility to use transitions properly.
  • WG-28549 Added AkPlatformInitSettings::bHwCodecLowLatencyMode to use low latency mode for XMA and ATRAC9 decoding.
  • WG-28789 Adjusted AK::AddSecondaryOutput(), AkInitSettings, and AkOutputSettings to support Audio Device plug-in parameters. See the Important Migration Notes 2016.1 for more details.
  • WG-28938 Corrected initialization of default values in AK::MemoryMgr::PoolMemInfo.
  • WG-29203 ReplacedAK::IAkMixerInputContext::IsPositioningEnabled() by the more useful AK::IAkMixerInputContext::IsSpatializationEnabled().
  • WG-29510 Moved 'Mix...()' methods from AK::IAkMixerPluginContext to AK::IAkGlobalPluginContext. AK::IAkGlobalPluginContext is accessible via AK::IAkPluginContextBase::GlobalContext().
  • WG-29771 Added media ID to AkCallbackType::AK_Duration callback.
