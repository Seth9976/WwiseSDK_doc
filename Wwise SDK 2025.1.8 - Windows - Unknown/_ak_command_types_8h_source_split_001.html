612 /// Seeks inside all playing objects that are referenced in Play Actions of the specified Event.
613 ///
614 /// Notes:
615 /// - This works with all objects of the Containers hierarchy, including Music Segments and Music Switch Containers.
616 /// - There is a restriction with sounds that play within a continuous sequence. Seeking is ignored
617 /// if one of their ancestors is a continuous (random or sequence) container with crossfade or
618 /// trigger rate transitions. Seeking is also ignored with sample-accurate transitions, unless
619 /// the sound that is currently playing is the first sound of the sequence.
620 /// - Seeking is also ignored with voices that can go virtual with "From Beginning" behavior.
621 /// - Sounds/segments are stopped if <tt>position</tt> is greater than their duration.
622 /// - <tt>position</tt> is clamped internally to the beginning of the sound/segment.
623 /// - If the option "Seek to nearest marker" is used, the seeking position snaps to the nearest marker.
624 /// With objects of the Containers hierarchy, markers are embedded in wave files by an external wave editor.
625 /// Note that looping regions ("sampler loop") are not considered as markers. Also, the "add file name marker" of the
626 /// conversion settings dialog adds a marker at the beginning of the file, which is considered when seeking
627 /// to nearest marker. In the case of interactive music objects (Music Segments, Music Switch Containers, and Music Playlist Containers), user (wave) markers are ignored:
628 /// seeking occurs to the nearest segment cue (authored in the segment editor), including the Entry Cue, but excluding the Exit Cue.
629 /// - This method posts a command in the sound engine queue, thus seeking will not occur before
630 /// the audio thread consumes it (after a call to RenderAudio()).
631 ///
632 /// Notes specific to Music Segments:
633 /// - With Music Segments, <tt>position</tt> is relative to the Entry Cue, in milliseconds. Use a negative
634 /// value to seek within the Pre-Entry.
635 /// - Music segments cannot be looped. You may want to listen to the AK_EndOfEvent notification
636 /// in order to restart them if required.
637 /// - In order to restart at the correct location, with all their tracks synchronized, Music Segments
638 /// take the "look-ahead time" property of their streamed tracks into account for seeking.
639 /// Consequently, the resulting position after a call to SeekOnEvent() might be earlier than the
640 /// value that was passed to the method. Use <tt>AK::MusicEngine::GetPlayingSegmentInfo()</tt> to query
641 /// the exact position of a segment. Also, the segment will be silent during that time
642 /// (so that it restarts precisely at the position that you specified). <tt>AK::MusicEngine::GetPlayingSegmentInfo()</tt>
643 /// also informs you about the remaining look-ahead time.
644 ///
645 /// Notes specific to Music Switch Containers:
646 /// - Seeking triggers a music transition towards the current (or target) segment.
647 /// This transition is subject to the container's transition rule that matches the current and defined in the container,
648 /// so the moment when seeking occurs depends on the rule's "Exit At" property. On the other hand, the starting position
649 /// in the target segment depends on both the desired seeking position and the rule's "Sync To" property.
650 /// - If the specified time is greater than the destination segment's length, the modulo is taken.
651 ///
652 /// This command can fail for the following reasons:
653 /// - AK_InvalidParameter: \c eventID is invalid or relative position is not in the valid range.
654 /// - AK_IDNotFound: Event was not found, or \c gameObjectID is specified but not a registered game object.
655 ///
656 /// \sa
657 /// - <tt>AkCommand_SeekOnEvent</tt>
658 /// - <tt>AK::SoundEngine::SeekOnEvent()</tt>
659 /// - <tt>AK::SoundEngine::GetSourcePlayPosition()</tt>
660 /// - <tt>AK::MusicEngine::GetPlayingSegmentInfo()</tt>
661 struct AkCmd_SeekOnEvent
662 {
663  AkUniqueID eventID; ///< Unique ID of the event
664  union
665  {
666  AkTimeMs absolute; ///< Desired position where playback should restart, in milliseconds
667  AkReal32 relative; ///< Desired position where playback should restart, expressed in a percentage of the file's total duration, between 0 and 1.f (see note above about infinite looping sounds)
668  } position;
669  AkGameObjectID gameObjectID; ///< (optional) Associated game object ID; use AK_INVALID_GAME_OBJECT to affect all game objects
670  AkPlayingID playingID; ///< (optional) Specify the playing ID for the seek to be applied to. Will result in the seek happening only on active actions of the playing ID. Let it be AK_INVALID_PLAYING_ID to seek on all active actions of this event ID.
671  AkUInt8 isPositionRelative; ///< Determines whether the position should be interpreted as absolute (milliseconds) or relative (%).
672  AkUInt8 seekToNearestMarker; ///< If non-zero, the final seeking position will be made equal to the nearest marker (see note above)
673 };
