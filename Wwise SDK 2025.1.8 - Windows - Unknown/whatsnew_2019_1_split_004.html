API Changes

  • WG-39800 Enhanced our API by adding a function, AK::SoundEngine::ExecuteActionOnPlayingID(AkActionOnEventType in_ActionType, AkPlayingID in_playingID, AkTimeMs in_uTransitionDuration, AkCurveInterpolation in_eFadeCurve), which will execute the specified Action on the active content associated with the playing ID.
  • WG-39919 (UWP) Changed the Motion sink API. AK::GetDeviceIDFromGamepad() must be called to retrieve the correct motion device ID of a UWP gamepad.
  • WG-40362 Added several new APIs:
    • AK::SoundEngine::ReplaceOutput to allow for easily changing the device or active ShareSet for a given output device. (Also added new section to IntegrationDemo Sample to demonstrate its application in a simple "Options" menu.)
    • AK::SoundEngine::GetDeviceSpatialAudioSupport to easily determine if a given deviceId supports spatial audio.
    • AK::GetWindowsDeviceCount and AK::GetWindowsDevice APIs to simplify enumeration of IMMDevices on Windows for most common cases.
    • (Xbox One) AkPlatformInitSettings::bEnablespatialAudio property to permit OS-level enabling of Spatial Audio services during sound engine initialization.
  • WG-40535 Changed Spatial Audio's API:
    • Removed QuerySoundPropagationPaths because the information returned is largely redundant with QueryDiffractionPaths.
    • Added Room and Portal ID to AkDiffractionPathInfo so that propagation paths can be returned via QueryDiffractionPaths.
    • Added new function QueryWetDiffraction to return information about diffraction for a specific portal.
  • WG-40716 Exposed platform-specific information to plug-ins through new method AK::IAkGlobalPluginContext::GetPlatformContext().
