Fixes for Community-Reported Bugs

  • WG-68133 Fixed: (Mac Authoring) Lack of stability when changing an object name in the project hierarchy.
  • WG-79546 Fixed: Crash when reading the cache data for an audio waveform.
  • WG-81644 Fixed: (Acoustics) Calling SetPortal copies the Portal name string, even when unchanged, resulting in an unnecessary allocation.
  • WG-81680 Fixed: AK ASIO Output plug-in is not allowed to retry device initialization after a recoverable initialization error.
  • WG-81837 Fixed: Crash when reading a .wem file that has a header spanning multiple read buffers, and the I/O Hook doesn't read a stream with consistent buffer sizes.
  • WG-82267 Fixed: Different converted audio files are generated depending on when the conversion takes place. If a file is edited in the Source Editor and then converted before closing the project, the converted file might differ from the converted file created when the project is closed between editing the file and conversion.
  • WG-82293 Fixed: ResetRTPCValueWithTransition API call displays wrong transition time in Capture Log.
  • WG-82335 Fixed: (Acoustics) Feedback loop between Room aux busses.
  • WG-82477 Fixed: SoundBank Generation log contains messages referring to missing Auto-Defined Soundbanks.
  • WG-82479 Fixed: SoundBank Generation log reports media duplication for Auto-Defined Soundbanks.
  • WG-82646 Fixed: Hang in AK::StreamMgr when using offline rendering.
  • WG-82680 Fixed: Erroneous "Media Not Found" error when generating Auto-Defined SoundBanks with Effect plug-ins on an audio bus.
