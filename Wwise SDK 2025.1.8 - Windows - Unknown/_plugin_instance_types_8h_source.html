105 /**
106  * \brief Plug-in frontend instance.
107  *
108  * A frontend contains the visual part of the Authoring plug-in. It is optional: for example, it is never instantiated when
109  * connecting through WwiseConsole. Therefore, do not implement any processing or business work in a frontend part.
110  *
111  * For example, avoid using the frontend for handling licensing, SoundBank generation, audio processing, properties (such as
112  * validating ranges for property sets), media handling, media conversion, custom data loading, or other related work.
113  *
114  * \aknote In legacy plug-ins from before Wwise 2021.1, there is only one instance of the Authoring plug-in that contains both
115  * the backend and the frontend. This has changed for multiple reasons: to be able to instantiate a plug-in backend without any
116  * frontend, or alternatively, to instantiate multiple copies of a frontend for a unique backend. \endaknote
117  */
118 struct ak_wwise_plugin_frontend_instance AK_WWISE_PLUGIN_DERIVE_FROM_INSTANCE_BASE {};

122  * \brief Define an instance type as a backend.

124  * In C++, this derives the backend instance from the ak_wwise_plugin_backend_instance in a separate action so that RTTI can help
