Bug Fixes

  • WG-68970 Fixed: Unreal brushes containing unaligned edge triangles (T-junctions) result in false positive boundary edges, causing a "Geometry is not watertight" warning and preventing the geometry from being used as a room.
  • WG-69595 Fixed: (Spatial Audio) Paths can escape through Portals with gaps between opening surfaces.
  • WG-71103 Fixed: (Spatial Audio) Distance is incorrectly calculated for parents of Reverb Zones.
  • WG-71800 Fixed: (Spatial Audio) Side-by-side Portals use the wrong diffraction point.
  • WG-71929 When using a high DPI monitor, clicking an object in the Project Explorer sometimes does not result in it being displayed in the Property Editor.
  • WG-72811 Fixed: Performance Monitor statistics in the Average column show "-nan %" in some scenarios.
  • WG-72990 Fixed: Voice Inspector displays voice contributions from parent nodes on properties that are overridden, for example Output Bus Volume.
  • WG-73058 Fixed: Wrong row can be selected when clicking a heavily populated Capture Log.
  • WG-73129 Fixed: (Spatial Audio) Spatialization of reverbs is incorrect because of a misinterpretation of AkInitSettings::fGameUnitsToMeters.
  • WG-73161 Fixed: Possible crash when trying to use the shortcut menu for a Music Segment.
  • WG-73196 Fixed: Music Stinger Play At property values Next Beat and Next Bar are reversed.
  • WG-73231 Fixed: (Spatial Audio) When the listener is transitioning between rooms, the distance to the outside room is sometimes returned as a very large number, resulting in it being inaudible.
  • WG-73299 Fixed: Game Parameter Simulation Value of 0 is reset to default after project reload.
  • WG-73329 Fixed: Music Playlist Item does not allow setting Avoid Repeat to zero.
  • WG-73372 Fixed: With Play Originals enabled, and with a very small Trim Start value, sound playback might play from the beginning instead of the specified value.
